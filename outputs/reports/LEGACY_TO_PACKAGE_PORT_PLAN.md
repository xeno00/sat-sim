# Legacy-To-Package Port Plan

## Executive Summary
Port estimator behavior before rerunning final manuscript figures; do not port truth-gated decisions.

| Technique | Legacy behavior | Why it helped | Defensible? | V24 port | Tests required |
|---|---|---|---:|---|---|
| clockless IL/GN preconditioning | Initialize UE positions with reduced/clockless equations before the full-clock solve. | Keeps the nonlinear full-clock solve from starting too far from the geometry. | True | Implement as a package-native Step 1 initializer using measurement residual cost only, then lift into the gauged V24 theta vector. | hand-computed DL-only initialization fixture; no truth-state access; improves or does not worsen residual in deterministic toy cases |
| full-clock internal solve with gauge-relative reporting | Solve an all-clock symbolic state, then report position/synchronization errors after post-processing. | The optimizer had enough internal degrees of freedom to absorb clock biases. | True | Use the explicit V24 gauged parameter vector internally, not all-clock state; report clocks relative to the reference satellite. | parameter order test; reference satellite column absent; sync metric excludes reference satellite |
| pseudoinverse / rank-tolerant updates | Use rank-tolerant linear algebra and dependent-row handling in difficult geometries. | Prevents brittle failures in underconstrained or nearly singular intermediate stages. | True | Allow rank-tolerant damped updates as diagnostic/non-successful initializers; do not report finite CRLB unless subspace estimability is proven. | rank-deficient status test; finite CRLB only for full rank; damped update never mislabeled as convergence |
| LM damping behavior | Use LM-style damping in the full nonlinear solve. | Stabilizes nonlinear TOA residual minimization compared with raw Gauss-Newton. | True | Package-native LM with accepted-step criteria based on weighted residual decrease and bounded step norms. | weighted residual decreases on accepted steps; damping adapts deterministically; no manuscript figure generated by unit tests |
| truth-gated acceptance replacement | Some LM/MAP acceptance or fallback decisions compare against true state error. | Can suppress bad-looking estimates in replay plots, but leaks oracle information. | False | Replace with observable criteria: weighted measurement residual, prior innovation, covariance trace, and consistency gates. | grep/no true-state access in estimator; acceptance depends only on z/h/R/P/Q; synthetic failure fallback is logged |
| MAP/global fallback behavior | Fallback paths keep a usable trajectory when a solver stage fails. | Avoids catastrophic rows dominating small Monte Carlo grids. | True | Implement explicit fallback hierarchy with status flags; include failed/fallback rows in diagnostics instead of hiding them. | fallback count test; failure log test; summary preserves non-success statuses |
| display smoothing/fitting separation | Plotting cells smooth or manually transform displayed curves. | Improves visual readability but can obscure raw algorithm behavior. | True | Keep raw CSV/NPZ as the authority; any smoothing must be a separate display transform recorded in metadata. | raw/display arrays both saved; metadata records transform; no smoothing in algorithm metrics |
